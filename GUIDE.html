1. 처음이라면 (5분 요약)

컴퓨터를 켜고 아무것도 모르는 상태라고 가정하고, 순서만 따라 하세요.

  1. Claude Code를 켭니다. (없으면 claude.com/claude-code 에서 먼저 설치)
  2. 화면 맨 아래 입력칸에 아래 3줄을 한 줄씩 붙여넣고 매번 Enter: 
    /plugin marketplace add sodam-ai/SoDam-Persona
/plugin install sodam-persona@sodam-persona
/reload-plugins
  3. Claude Code를 완전히 껐다가 다시 켭니다. (아주 중요 — 이래야 켜집니다)
  4. 새 대화에서 그냥 평소처럼 질문하면 됩니다. 예: 이 부가세 신고 어떻게 해 → AI가 세무 전문가처럼 답하고 끝에 "세무사 확인 권장" 안내가 붙으면 정상입니다.

💡 이게 전부입니다. 어려운 명령을 외울 필요 없이, 평소 말로 질문하면 페르소나가 알아서 적용됩니다.

2. 자주 묻는 질문 (FAQ)

Q. 설치하면 내 컴퓨터에 뭐가 바뀌나요? A. Claude Code의 플러그인 목록에 sodam-persona이 추가되고, 새 세션부터 AI에게 성격·규칙이 자동 주입됩니다. 다른 프로그램이나 파일은 건드리지 않습니다.

Q. 인터넷이 꼭 필요한가요? A. 설치(다운로드)할 때만 필요합니다. 설치 후 페르소나 작동 자체는 로컬(내 컴퓨터)에서 돌아갑니다.

Q. 비밀번호나 API 키가 들어 있나요? 공개돼도 안전한가요? A. 페르소나 텍스트(성격·규칙)만 있고 비밀정보는 없습니다. 그래서 공개(Public) 저장소로 둬도 안전합니다. (실제로 비밀정보 스캔에서 0건 확인)

Q. 토큰(요금)이 더 드나요? A. 항상 켜지는 부분은 매우 가볍습니다. 무거운 성격 본문은 세션 시작 때 한 번 주입됩니다. "투자/세무/보안" 같은 스킬은 그 주제가 나올 때만 불러와 그때만 토큰을 씁니다.

Q. 다른 플러그인과 충돌하나요? A. 페르소나는 "외부 플러그인의 자동 스킬 권유는 무시"하도록 되어 있지만, 자기 자신의 스킬(persona-*)은 예외로 자동 활성합니다. 그래서 충돌 없이 페르소나가 우선합니다.

Q. 끄고 싶으면요? A. 잠깐: 대화에서 그냥 답해 또는 페르소나 끄고. 완전히: claude plugin disable sodam-persona@sodam-persona 후 재시작.

Q. 여러 대의 컴퓨터에서 똑같이 쓰려면? A. 각 컴퓨터에서 README 4번 3줄을 그대로 따라 하면 동일하게 적용됩니다.

Q. 세무·법률 답변을 그대로 믿어도 되나요? A. 아니요. 이 페르소나의 세무·법률 답변은 일반 정보 제공용이며, 답변에도 "전문가 최종 확인 권장" 문구가 자동으로 붙습니다. 실제 신고·계약 전에는 반드시 세무사·변호사 등 전문가의 확인을 받으세요.

Q. 상업적으로 사용해도 되나요? A. 네. Apache-2.0 라이선스라 상업적 사용이 허용됩니다. 단 저작권·라이선스 고지를 유지하고, 파일을 바꿨으면 바꿨다고 표시해야 합니다(8번).

3. 상황별 문제 해결 레시피

🔴 "재시작했는데 페르소나가 안 켜진 것 같아요"

  1. 설치·활성 확인: 
    claude plugin list
    • 목록에 없다 → 설치 안 됨 → README 4번
    • disabled → claude plugin enable sodam-persona@sodam-persona
  2. 구성요소 인식 확인: 
    claude plugin details sodam-persona
    • Skills (7) + Hooks (2) 가 보여야 정상.
  3. 완전히 종료 후 재실행 (창만 닫지 말고 프로그램 자체를 종료).
  4. 새 세션에서 객관적으로 깊게 봐줘 → 페르소나 반응 확인.

🔴 "install 했더니 'Marketplace not found' 라고 떠요" (자주 발생)

원인: 예전에 제거(Uninstall) 할 때 마켓플레이스 등록까지 사라졌기 때문입니다. 해결 — 등록부터 다시:

/plugin marketplace add sodam-ai/SoDam-Persona
/plugin install sodam-persona@sodam-persona
/reload-plugins

👉 항상 add(등록) → install(설치) → reload(적용) 순서.

🔴 "내용을 고쳤는데 그대로예요" (가장 흔함)

원인: 설치본은 캐시(복사본) 입니다. 원본 폴더만 고치면 캐시는 옛날 그대로입니다. 순서대로:

claude plugin marketplace update sodam-persona
claude plugin uninstall sodam-persona@sodam-persona
claude plugin install sodam-persona@sodam-persona

그 후 재시작. (확인: 캐시 폴더 파일이 수정 내용을 담았는지 봐도 됩니다.)

🔴 "GitHub에서 바로 설치가 안 돼요"

  • 저장소 이름을 정확히: sodam-ai/SoDam-Persona (대소문자 포함).
  • 인터넷 연결 확인. 회사·학교 네트워크는 GitHub 접근이 막혀 있을 수 있습니다.
  • 그래도 안 되면 다운로드 후 로컬 설치: 저장소를 clone 하거나 ZIP으로 받아 압축을 풀고, 그 폴더 경로로 /plugin marketplace add (폴더경로).

🟡 "uv: command not found 에러가 떠요"

  • 이건 다른 플러그인의 Stop 훅이 uv(파이썬 도구)를 못 찾아 나는 것으로, 이 페르소나 플러그인과 무관합니다(이 플러그인엔 Stop 훅이 없음). 페르소나 작동엔 영향 없습니다.

🟡 "설정이 통째로 안 먹는 느낌이에요"

  • settings.json은 JSON이 한 글자라도 틀리면 그 파일 설정 전체가 무효가 됩니다.
  • 형식 검사 후, 깨졌으면 백업에서 복원(README 16번).

4. 유지보수: 내용을 고치는 법

성격·규칙을 바꾸려면 → sodam-persona/hooks/persona_core.md 수정 (항상 켜지는 정본). 트리거 단어를 추가하려면 → 같은 persona_core.md의 트리거 목록에 추가 (정본은 코어 한 곳). 특정 전문가 상세를 바꾸려면 → sodam-persona/skills/persona-(이름)/SKILL.md 수정.

수정 후 항상 이 순서를 지키세요:

# 1) 저장소 폴더에서 파일 수정

# 2) 정합성 검사 (권장 — 관점 수·스킬 수·면책 존재 자동 확인)
node validate.mjs

# 3) 캐시에 반영
claude plugin marketplace update sodam-persona
claude plugin uninstall sodam-persona@sodam-persona
claude plugin install sodam-persona@sodam-persona

# 4) Claude Code 재시작

# 5) GitHub에 올리기 (⚠️ 바꾼 파일만 지정해서 add — 'git add -A'는 쓰지 마세요)
git add sodam-persona/hooks/persona_core.md   # 예시: 실제 바꾼 파일들만
git commit -m "fix: 수정 내용 요약"
git push

⚠️ 왜 git add -A를 피하나요? 폴더 안의 의도치 않은 파일(다른 도구가 남긴 임시 파일 등)까지 통째로 올라가 공개 저장소를 오염시킬 수 있습니다. 바꾼 파일만 이름을 적어 add 하세요. ⚠️ 캐시 반영을 빼먹지 마세요. 위 3단계를 건너뛰면 예전 버전이 계속 켜집니다(가장 흔한 착각).

5. 작동 원리 (조금 더 깊게)

  • SessionStart 훅 (inject-core.js): 세션이 시작될 때 persona_core.md를 읽어 AI의 맥락(context)에 통째로 넣습니다. → 성격·강도(L0~L3)·15관점·트리거·면책 규칙이 항상 살아 있음.
  • UserPromptSubmit 훅 (inject-marker.js): 입력을 보낼 때마다 persona_marker.txt를 넣습니다. → 컴팩션(대화 압축)·서브에이전트 뒤에도 페르소나가 즉시 복구되고, 7가지 자가점검과 면책 강제가 매번 적용됨.
  • 스킬(skills 7개): "투자/세무/보안/디자인" 같은 단어가 감지되면 해당 스킬(상세 지식)을 불러옵니다. 트리거 판단의 정본은 코어, 스킬은 그 확장 상세만 담습니다(중복·어긋남 방지).
  • 면책 강제: 회계·세무(#14)·법률(#11) 도메인의 실행성 답변(신고·금액계산·조항해석·판단)에는 "전문가 최종 확인 권장" 문구가 반드시 포함됩니다. 이 규칙이 조건부 스킬이 아니라 항상 켜지는 코어·마커에 들어 있어, 스킬 로드 여부와 무관하게 안정적으로 작동합니다.
  • 경로 방식: 훅 스크립트는 __dirname(자기 위치 기준)으로 데이터를 읽어, 새 PC에서 경로가 달라도 안 깨집니다. 이게 "한 번에 이식"의 핵심입니다.

6. 검증하는 법 (제대로 깔렸는지)

# 1) 플러그인 형식 검사
claude plugin validate (저장소경로)/sodam-persona

# 2) 설치·인식 확인
claude plugin list
claude plugin details sodam-persona   # Skills (7) + Hooks (2) 확인

# 3) (개발자용) 정합성·면책 자동 검사
node validate.mjs                      # 관점 15 · 패턴 20 · 스킬 7 · 면책 존재 확인
  • 실제 작동은 재시작 후 대화로 확인합니다:
    • 트리거 없이 평소 질문 → 페르소나 톤(신중·근거 중심)
    • 이 거래 검토해줘 → 투자자 스킬 자동 로드
    • 이 부가세 신고 봐줘 → 세무 스킬 + 끝에 면책 문구 확인 (이게 나오면 최신 버전 정상)

7. 용어 사전

용어 뜻
컨텍스트(context) AI가 한 번에 기억하는 대화·정보의 범위
컴팩션(compaction) 대화가 길어지면 자동으로 요약·압축하는 것 (마커가 페르소나를 복구)
훅(hook) 특정 시점에 자동 실행되는 명령
스킬(skill) 필요할 때 불러오는 지식/지침 묶음
마켓플레이스(marketplace) 플러그인 목록(창고)
캐시(cache) 빠르게 쓰려고 복사해 둔 사본 (설치본이 여기 있음)
JSON 설정을 적는 형식. 괄호·쉼표 하나만 틀려도 전체가 깨짐
L0~L3 답변 강도 단계 (L0 잡담 → L3 보안·돈·배포 같은 중대 작업)
면책(disclaimer) "참고용이며 전문가 확인 권장" 같은 책임 한계 안내 문구
트리거(trigger) 특정 단어가 나오면 관련 관점·스킬을 켜는 방아쇠
